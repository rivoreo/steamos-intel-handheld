A SteamOS package path with hardware context.

SteamOS is Arch-based, but Intel handheld support needs more than a package file. It needs the right system integration, the right sensor permissions, and a verifier that proves the device actually works.

01

What it is

A SteamOS support layer for Intel handheld PCs, starting with SteamOS Manager TDP control and MangoHud power sensor access.

02

What it can do

Install a remote TDP provider, prepare readable RAPL power counters, host pacman repo paths, and verify the result on real hardware.

03

Why it exists

Intel handhelds need SteamOS integration that survives normal use without replacing SteamOS Manager or inventing a parallel telemetry stack.

04

How to install

Add the future `rivoreo-steamos` pacman repository or run the bootstrap command once signed packages are published.

What the repository owns.

The project keeps device support narrow and inspectable: system service assets, install paths, packaging, and verification live together.

SteamOS Manager TDP remote

Exports the SteamOS Manager `TdpLimit1` remote interface through a system service.

Intel RAPL power path

Maps SteamOS TDP requests to Intel RAPL limits and exposes package power counters.

MangoHud sensor access

Prepares CPU and GPU power sensors so MangoHud can read existing Linux counters.

Pacman repository hosting

Reserves a signed Arch-style package repository at `holo.libz.so` for SteamOS installs.

QEMU build environment

Builds SteamOS-compatible `mangoapp` artifacts against Valve's recovery image userland.

Hardware verification harness

Checks TDP state, RAPL limits, sensor readability, SteamOS Manager discovery, and failed units.
